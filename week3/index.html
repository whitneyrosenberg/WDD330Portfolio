Object Methods: this
  • For an object method to access the information stored in the object the method can use the this keyword.
  • In JavaScript this is "free". It's value is evaulated at call-time and does not depend on where the method was declared, but rather on what object is "before the dot."
  • Arrow functions don't have their "own" this. Using this in an arrow function will take it from the outer "normal" function

Objects
  • An object literal is an object that is created directly in the language by wrapping all its properties ad methods in curly braces.
  • If a property's name doesn't follow the rules for naming variables, it needs to e quoted.
  • All objects are mutable at any time when a program is running, even if it was declared using const.
  • Access youn objects properties using dot notation or bracket notation.
  • Symbols can be used as property keys. Using them as keys avoids any naming clashes.
  • The in operator can be used to check whether an object has a particular property.
  • Another way is to use the hasOwnProperty() method.
  • Loop through all of an object's properties and methods by using a for in loop.
  • The Object.keys() method will return an array of all of the keys of any object that is provided as an argument.
  • The Object.values() method works the same way, but returns an array of all of hte object's values. (ES2017)
  • The Object.entries() method returns an array of key-value pairs. (ES2017)
  • Use the delete operator to remove a property from an object.
  • Objects are assigned by reference. Changes made to any reference of the object will affect the same object. This does not happen with primitives.
  • Defining functions in a namespace can help avoid naming collisions.
JSON
  • JSON is a string representation of the object literal notation that we have just seen. There are, however, a couple of key differences:
    1. Property names must be double-quoted.
    2. Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects.
    3. Functions are not permitted values.
  • The parse() method takes a string of data in JSON format and returns a JavaScript object.
  • The stringify() method takes a JavaScript object and returns a string of JSON data.
Math Object
  • The Math object has eight properties that represent a mix of commonly used math constants.
    • Math.PI // The ratio of the circumference and diameter of a circle
    • Math.SQRT2 // The square root of 2
    • Math.SQRT1_2 // The reciprocal of the square root of 2
    • Math.E // Euler's constant
    • Math.LN2 // The natural logarithm of 2
    • Math.LN10 // The natural logarithm of 10
    • Math.LOG2E // Log base 2 of Euler's constant
    • Math.LOG10E // Log base 10 of Euler's constant
  • The Math.abs() method returns the absolute value of a number.
  • The Math.ceil() method will round a number up to the next integer.
  • The Math.floor() method will round a number down to the next integer.
  • The Math.round() method will round a number to the nearest integer.
  • The Math.truc() method will return the integer part of a number. (ES6)
  • The Math.exp() method will raise a number to the power of Euler's constant.
  • The Math.pow() method will raise any number to the power of another number.
  • The Math.sqrt() method returns the positive square root of a number.
  • The Math.cbrt() method returns the cube root of a number. (ES6)
  • The Math.hypot() method returns the square root of the sum of the squares of all its arguments. It can be used to calculate the hypotenuse of a right-angled triangle. (ES6)
  • The Math.max() method returns the maximum number from its arguments.
  • The Math.min() method returns the minimum number from the given arguments.
  • The Math object has the standard trigonometric functions. All angles are measured in radians for these functions.
  • The Math.random() method is used to create random numbers.
The Date Object
  • Create a new Date object with the constrcutor and the new operator. If no argument is supplied, the date will default to the current date and time.
  • When supplying a date for an argument, remember that January is 0, not 1, and Sunday is 0, not 1.
  • The Date object has getters and setters available.
The RegExp Object
  • The preferred way to create a regular expression is to use the literal notation of writing the regular expression between forward slashes.
  • You can also create an new instance of the RegExp object using the new operator and a constructor function.
  • Use the test() method to see if a string matches the regular expression pattern.
  • The exec() method works in the same way but instead of returning a boolean it returns and array containing the first match found or null if there are no matches.
    • Special Characters
  • . matches any character, except line breaks
  • \w matches any word character, and is equivalent to [A-Za-z0-9_]
  • \W matches any non-word character, and is equivalent to [\^A-Za-z0-9_]
  • \d matches any digit character, and is equivalent to [0-9]
  • \D matches any non-digit character, and is equivalent to [^0-9]
  • \s matches any whitespace character, and is equivalent to [ \t\r\n\f]
  • \S matches any non-whitespace character, and is equivalent to [^ \t\r\n\f]
    • Modifiers
  • Modifiers can be placed after a token to deal with multiple occurrences of that token.
  • ? makes the preceding token in the regular expression optional
  • * matches one or more occurrences of the preceding token
  • + matches one or more occurrences of the preceding token
  • {n} matchesnoccurrences of the preceding token
  • {n,} matches at leastnoccurrences of the pattern
  • {,m} matches at mostmoccurrences of the preceding token
  • {n,m} matches at leastnand at mostmoccurrences of the preceding token
  • ^ marks the position immediately before the first character in the string
  • $ marks the position immediately after the last character in the string

    • The recommended way of dealing with events, and the current standard, is to use event listeners.
    • The addEventListener() method is called on a node object to attah an event listener to the document's body.
    • Three events that occur when a user presses a key are: keydown, keypress, and keyup.
    • The touchstart event occurs when a user initially touches the surface.
    • The touchend event occurs when a user stops touching the surface.
    • The touchmove event occurs after a user has touched the screen then moves around without leaving.
    • The touchenter event occurs when a user has already started touching the surface, but then passes over the element to which the event listener is attached.
    • The touchleave event occurs when the user is still touching the surface, but leaves the element to which the event listener is attached.
    • The touchcancel event occurs when a touch event is interrupted, such as a user’s finger moving outside the document window, or too many fingers being used at once.
